# Diagram

Render the current architectural picture for a feature. Read-only utility: collects Mermaid blocks from the feature's existing design artifacts into a single consolidated `docs/{feature}/diagrams.md`. No sub-agents; no gate; target wall-clock <10s.

Use when you want to see "the shape of the feature as it stands right now" without triggering `/design` re-entry, Phase 3b drafting, or an architect gate. Parallel to `/status` (portfolio-level snapshot) — this is the feature-level equivalent.

## Phase 0: Parse arguments

- `$ARGUMENTS[0]` — feature slug (required). Reject if missing: "Usage: /diagram <feature>".

## Phase 1: Verify prerequisites

`docs/{feature}/` MUST exist with at least `map.md` OR `design/README.md`. If neither:

```
docs/{feature}/ has neither map.md nor design/README.md. Nothing to render.
Run /map {feature} first.
```

Stop.

## Phase 2: Collect Mermaid blocks

Read each of the following, extracting every fenced ```mermaid``` block plus its surrounding H2/H3 heading as the block title:

| Source file | What to extract | Section label |
|---|---|---|
| `docs/{feature}/map.md` | Flowchart in "Entry points and data flow" | "Map — data flow (from map.md)" |
| `docs/{feature}/design/README.md` | `## Module sketch` block | "Scope — module sketch (Gate 1)" |
| `docs/{feature}/design/README.md` | `## Data-flow sketch` block | "Scope — data-flow sketch (Gate 1)" |
| `docs/{feature}/design/01-architecture.md` | Module-structure Mermaid | "Architecture — module structure" |
| `docs/{feature}/design/01-architecture.md` | `## Delta view` Mermaid | "Architecture — delta view (what changes)" |
| `docs/{feature}/design/02-behavior.md` | Data-flow + sequence diagrams | "Behavior — data flow / sequences" |
| `docs/{feature}/design/04-data-flow.md` (if present) | All Mermaid blocks | "Data flow (detail)" |

For every source file that exists but contains no Mermaid block, record a line for the "Missing" section.

## Phase 3: Write diagrams.md

Output path: `docs/{feature}/diagrams.md`. Overwrite if present.

Format:

```markdown
---
feature: {slug}
generated: YYYY-MM-DD
regenerated-from:
  - docs/{feature}/map.md                        (sha: <first-12-chars of git blob hash or file mtime-epoch>)
  - docs/{feature}/design/README.md              (sha: ...)
  - docs/{feature}/design/01-architecture.md     (sha: ...)
  - docs/{feature}/design/02-behavior.md         (sha: ...)
---

# Diagrams — {feature}

Auto-generated by `/diagram {slug}`. Do not hand-edit — this file is overwritten every run.

## Scope — module sketch (Gate 1)

{quoted Mermaid block from design/README.md}

## Scope — data-flow sketch (Gate 1)

{quoted Mermaid block from design/README.md}

## Architecture — module structure

{quoted Mermaid block from design/01-architecture.md}

## Architecture — delta view (what changes)

{quoted Mermaid block from design/01-architecture.md § Delta view}

## Behavior — data flow / sequences

{quoted Mermaid blocks from design/02-behavior.md — one ### subsection per block}

## Map — data flow (from map.md)

{quoted Mermaid block from map.md}

## Missing

Sources that were expected but had no Mermaid block (or the source file did not exist):

- `docs/{feature}/design/01-architecture.md § Delta view` — not found (feature pre-dates MADR-H2, or this section wasn't drafted)
- ...

If everything was present, write "Nothing missing — all expected sources had Mermaid blocks."
```

`regenerated-from` can use `stat -c '%Y' <path>` (mtime-epoch) if no cheap hash is available; the point is to let a follow-up `/diagram` detect whether any source changed.

## Phase 4: Surface the output

```
Diagrams rendered at docs/{feature}/diagrams.md ({N} blocks, {M} sources).

Missing:
  {quote the missing list, or "none"}
```

Do NOT recommend a next command. This is a utility; the caller already knew what they wanted.

## Rules

1. **Read-only.** Writes exactly one file: `docs/{feature}/diagrams.md`. Never edits source design docs.
2. **No sub-agent dispatch.** Main agent only. If rendering takes longer than ~10s you're doing it wrong — this is grep + concatenate, not analysis.
3. **No ADRs, no prose.** Pure collect-and-paste. If a source diagram is cryptic, the fix is in the source file, not here.
4. **No auto-regeneration of source.** If a source file is missing a Mermaid block, report it in the Missing section and move on. Do not invoke `/design` or draft diagrams ad-hoc.
5. **Overwrite without warning when frontmatter hashes match.** If a user hand-edits `diagrams.md` between runs, their edits are lost on the next `/diagram` — that is the contract. Hand-edits belong in the source design docs.
